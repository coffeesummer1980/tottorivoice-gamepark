# とっとりVOICE スイカゲーム風プロジェクト 要件定義書

## 1. ゲームの目的
プレイヤーは「とっとりVOICE」のキャラクターたちをボックスの中に落とし、同じキャラクター同士を合体（シンカ）させて、より大きなキャラクターへと成長させることを目指します。
最終目標は、最も大きなキャラクター（「スイカ」に相当する存在）を作ること、およびハイスコアを更新することです。

## 2. 想定プレイ時間
- **1プレイ:** 3分 〜 10分程度
- 隙間時間に遊べるカジュアルさを重視

## 3. 対象ユーザー
- **年齢層:** 子供から大人まで全年齢
- **ITリテラシー:** 不問（直感的な操作のみ）
- 「とっとりVOICE」のファン層、およびパズルゲーム好き

## 4. ゲームルール（文章）
1. プレイヤーは画面上部からキャラクターを一つずつフィールド（箱）に落とします。
2. 物理演算により、キャラクターは転がったり積み重なったりします。
3. **同じ種類のキャラクター**同士が接触すると、ポン！と弾けて、**一段階上の大きさのキャラクター**に進化します。
4. 進化するごとにキャラクターは大きくなり、スコアが加算されます。
5. 箱からキャラクターがあふれて、上部のデッドラインを超えて一定時間留まるとゲームオーバーです。

### キャラクター進化フロー（仮）
（※具体的な画像素材は実装時に割り当てます。全11段階程度を想定）
1. 極小キャラ（例: さくらんぼ級）
2. 小キャラ（例: いちご級）
3. ...
4. ...
11. 最大キャラ（例: スイカ級 - とっとりVOICEロゴや集合絵など）

## 5. 勝敗条件
- **勝ち（目標）:** 最大レベルのキャラクターを作成する（ただしゲームは続く）。
- **負け（終了）:** 重なったキャラクターが箱の上限ライン（デッドライン）を越える。

## 6. 操作方法（スマホ・PC両対応）
### スマートフォン（タッチ操作）
- **タップ/ドラッグ:** キャラクターの投下位置を左右に移動
- **指を離す:** キャラクターを落とす

### PC（マウス操作）
- **マウス移動:** キャラクターの投下位置を左右に移動
- **クリック:** キャラクターを落とす

## 7. 必須要素（MVP: Minimum Viable Product）
1. **物理演算エンジン:** Matter.js等を使用し、自然な挙動（転がる、落ちる、弾む）を実現。
2. **キャラクター表示:** 既存の画像素材（`画像/`フォルダ内のPNG/JPEG）を丸型にマスクして使用。
3. **合体ロジック:** 同種接触 -> 進化 -> スコア加算。
4. **ゲームオーバー判定:** デッドライン越えの検知。
5. **スコア表示:** 現在のスコア常時表示。
6. **「次」のキャラ表示:** 次に落とすキャラクターのプレビュー（NEXT）。
7. **レスポンシブ対応:** スマホ縦画面、PCブラウザでの適切な表示サイズ調整。

## 8. 今回やらないこと（スコープ外）
- **サーバーサイド開発:** ユーザー認証、オンラインランキング機能は実装しない。（ローカルストレージにハイスコア保存は可）
- **複雑な演出:** 派手すぎるパーティクルや3D演出。（動作軽量化を優先）
- **BGM/SE:** 必須ではないが、素材があれば実装。今回は仕様には含めない。
- **SNSシェア機能:** MVP段階では必須としない。

## 9. 将来拡張できそうなアイデア
- **オンラインランキング:** Firebase等を使った世界ランキング。
- **キャラボイス:** 合体時に「やったー！」「あらあら」などのボイス再生（とっとりVOICEの強みを活かす）。
- **スキン変更:** 季節ごとの衣装チェンジ。
- **特殊アイテム:** 邪魔なキャラを消せるアイテムなど。
