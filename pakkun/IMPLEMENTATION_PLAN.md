# ゆかりんぱっくんゲーム（キャッチゲーム）実装計画

## 1. ゲーム概要
空から降ってくる「鳥取の特産品（食べ物）」を、ゆかりんを操作してパクパク食べる（キャッチする）ゲームです。

## 2. 実装機能
- **プレイヤー（ゆかりん）**:
  - 画面下部を左右に移動（マウス、タッチ、キーボード）。
  - 通常時は「普通.png」または「yukarin.png」。
  - アイテムキャッチ時に「食べる.png」→「おいしい/ニコ」などの表情変化アニメーション。
  - ミス時（爆弾などを取った場合、または食べ物を落とした場合）に「やっちゃった.png」や「くたくた.png」。
- **アイテム（落下物）**:
  - `tottorifood`フォルダ内の画像を使用。
  - ランダムな位置から落下。
  - 落下速度は徐々に上がる、またはアイテムによって異なる。
- **ゲームシステム**:
  - **スコア**: 食べたアイテムに応じて加算。
  - **制限時間**: あり（例: 60秒）または「ライフ制（3回ミスで終了）」。
  - **ゲームオーバー**: リザルト画面でスコア表示。「やったー.png」などで祝福。

## 3. ファイル構成
- `index.html`: ゲーム画面、キャンバス、UI。
- `style.css`: 「とっとりVOICE」らしい明るくポップなデザイン。
- `game.js`: ゲームロジック（Canvas描画、物理判定、状態管理）。
- `assets.js`: 画像パスの定義（日本語ファイル名のエンコード管理）。
- `../game-park-sdk.js`: 共通SDK（レベル管理、コイン、ステータスバー表示）。

## 4. 開発ステップ
1. **基盤作成**: HTML/CSS/JSファイルの作成と画像の読み込みテスト。
2. **プレイヤー実装**: ゆかりんの表示と左右移動。
3. **アイテム実装**: 上からアイテムが降ってくるロジック。
4. **判定実装**: 当たり判定（キャッチ）とスコア加算、表情変化。
5. **ゲームサイクル**: タイトル画面、プレイ中、ゲームオーバー画面の遷移。
6. **仕上げ**: デザイン調整、演出強化（パーティクルなど）。

## 5. 使用アセット
- **プレイヤー**: `pakkun/ゆかりんキャラ/`
  - 待機: `普通.png`
  - キャッチ: `食べる.png`
  - 喜び: `にこ.png` / `嬉しい.png`
  - ダメージ/ミス: `やっちゃった.png` / `怒る.png`
- **アイテム**: `pakkun/tottorifood/` (5種類)
